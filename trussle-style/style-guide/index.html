How to Use the Styles

Developers have some options on how to attach these styles to their HTML (in order of preference):

  1. Include the SASS and use mixins (eg. @include tsl-x).
  2. Include trussle-style.css and use CSS classes (.tsl-x).
  3. Add tsl-style to a top-level element and use semantic elements (.tsl-style x).

Some features require Javascript; these features have instructions on how to set them up in their section.

Colours

The following colours can be used these ways (in order of preference):

  1. In SASS, use the variable $tsl-colour_name)
  2. In CSS, use the classes .tsl-colour_name for foreground or .tsl-bg-colour_name for background.

Layout

The Grid

The basis of all designs is the 12-column grid. Layouts are crafted using this, and elements are aligned along the columns. In order to maintain clean, consistent and considered layouts throughout, we should have margin at the top and the sides of the grid.

The margin is flexible based on the screen and device. Larger screens may require more margin, with mobile screens the smallest margin as they require more real estate for the interface. We use the defaults from Foundation here:

On screens less than 640px, we use a 12px border.
On screens between 640px and 1024px, we use a 24px border.
On screens higher than 1024px, we keep the page at 1024px and center it on the page.

No assistance is given for the grid in this package. Foundation Grid is used in most projects to provide the grid system.

Spacing

UI elements are sized horizontally to the grid, they may be between 1 and 4 columns wide and spaced between themselves according to the grid margins.

Paragraphs of copy should be restrained in width to ensure line lengths are never more than 8-12 words.

All spacing is based on 8px multiples. Some SASS variables are given to help:

  • $tsl-space-multiple
    8px  
  • $tsl-space-tiny
    8px  
  • $tsl-space-small:
    16px  
  • $tsl-space-medium:
    24px  
  • $tsl-space-large:
    32px  
  • $tsl-space-huge:
    40px  

Creating Layouts

This short guide will give you a basic direction when creating new page layouts.

  • Content can run across the entire screen.
  • Questions always align in the same place.
  • Content can also span a limited amount of columns but is always ranged left.
  • On the more functional product pages, (e.g. Fact Find) text is usually left aligned to improve readability and layout. However, when suitable, copy can be centered. For example, a lightbox modal that is centered in the screen can have centered copy to match.
  • Only Source Sans Pro should be used within the product.

Structuring Additional Content

There will be situations where a parent question may lead to further secondary questions. Parent questions will always range left as per general layouts, secondary questions will range left but from an indentation, indicating that it is the child of the parent. This allows us to add in as much additional content as required.

  • When the parent question is short, the child questions can fill the space on the right of the screen.
  • In some situations the parent question will extend further over the page, here the secondary question will still align along the indentation, but underneath the parent.

Form Elements

Buttons

The type style is based on the body from the typography section, set in the same colour as the button (unless filled, in which case it is white).

This button style has a minimum width of 144px, or have a padding of 24px.

Primary Buttons are for the most prominent CTAs. Secondary Buttons are used as the second level for standard buttons throughout the interface. Tertiary Buttons are special style buttons, used for buttons that require additional attention but shouldn't be the most prominent on the page.

.active and .disabled classes are available to apply to buttons styled with .tsl-button.

Also, don't forget to use tabindex for accessibility.

Use with the mixin tsl-button-primary (SASS), .tsl-button-primary (CSS) or .tsl-style button.

Use with the mixin tsl-button-secondary (SASS), .tsl-button-secondary (CSS) or .tsl-style button.secondary.

Use with the mixin tsl-button-tertiary (SASS), .tsl-button-tertiary (CSS) or .tsl-style button.tertiary.

Entry Fields

Entry fields are based on the same style as the buttons. The entry field type is based on the body from the typography section.

Labels are semi-bold and placed either above or to the side of the input.

Use with the mixin tsl-input-primary (SASS), .tsl-input-primary (CSS) or .tsl-style input. Use .valid and .invalid to specify when a field is valid or not. (:invalid works, but :valid does not because it is applied over the top of the default.)

Use with .tsl-currency. Make sure to include the JavaScript for this input, currency.js.

The second style is used when completing statements.

Use with the mixin tsl-input-secondary (SASS), .tsl-input-secondary (CSS) or .tsl-style input.secondary

Dropdowns

Dropdown Style 1 is based on the entry field style.

Use with the mixin tsl-select-primary (SASS), .tsl-select-primary (CSS) or .tsl-style select.

The second dropdown style is used within a body of text that requires you to complete a statement.

This field must have a required attribute in order to show/hide the arrow. Use with the mixin tsl-select-secondary (SASS), .tsl-select-secondary (CSS) or .tsl-style select.secondary.

Expanding Fields

Using JavaScript and CSS selectors, inputs and dropdowns can be made to fit the selected option or entered text.

Use with input.tsl-expanding-input. Make sure to include the JavaScript for this feature, expanding-input.js.

Use with select.tsl-expanding-select. Make sure to include the JavaScript for this feature, expanding-select.js.

Labels

Labels are semi-bold and are placed either to the side of (default) or above the labelled input. Don't use colons (:) in the label, and make labels full questions.

This can be used in a number of ways:

  1. In SASS, you can style a label with the mixin tsl-label.
  2. In CSS, you can style a label directly with .tsl-label.
  3. Also in CSS, you can enclose a label.question and input.answer in an element styled with .tsl-question-group.
  4. Finally, also in CSS you can use a label within a .tsl-style element.

By default, labels will be to the side of the input, so we have to enclose the label and input in a .tsl-question-group.on-top to place the label above the input. In SASS, you can emulate this with the mixin tsl-question-group-on-top.

Checkboxes

The checkbox works by hiding the input and replacing it with a PNG version. A label must immediately follow this input.

Use one of the following:

  • Use the mixins tsl-checkbox on an input or tsl-checkbox-label on a label (SCSS).
  • .tsl-checkbox on an input, followed by a label (CSS).
  • .tsl-style input[type="checkbox"]followed by .tsl-style label.

Large Selection Boxes (Radio Buttons)

Within forms there will be multiple choice questions that require a one click answer. When there are more than 6 options they should be consolidated into a dropdown.

The radio buttons work by hiding the input and replacing it with a label. A label must immediately follow this input.

Use one of the following:

  • Use the mixins tsl-radio on an input and tsl-radio-label on a label (SCSS).
  • .tsl-radio on an input, followed by a label (CSS).
  • .tsl-style input[type="radio"]followed by .tsl-style label.

The tsl-radio mixin in SCSS can take an argument to specify the size of the radio buttons; by default, the button is 150px.

UI Elements

Card


Related content for a user


Actionable content for the user


A clickable card


Cards are designed to group related content for the user. Those that contain actionable content should use .tsl-card-primary; those with secondary information can use .tsl-card.

Any card that can be clicked by the user should be .tsl-card-clickable. Don't forget to add tabindex="0" to the card to ensure it can be clicked using the keyboard.

Lightbox

Lightboxes are used to store additional content for the user. They are designed to sit on top of the page below but sticking to the 12 column grid which the rest of the design sits on. At least one column should be allowed on each side of the lightbox as a margin.

The mobile version is designed to sit on top of the page below but sticking to the 6 column grid which the rest of the design sits on. Lightboxes span full screen on mobile.

Text can be left aligned or centered, depending on the type of content, but line lengths should always be 8-12 words.

No lightbox assistance is given in this package. Foundation Reveal Modal is used in most projects to provide lightboxes.

Photography, Illustration & Iconography

Brand guidelines and resources are available here (Trussle only).

Photography

The Trussle photography style is candid, natural and should always feel like it’s telling the story of a real person, not staged stock photography. We use photography to talk about real people, mostly when referring to real customers, the team or the culture.

It should have a focus on space (rooms, skyline) and ownership (doors, DIY), on the excitiment and capturing the moment of getting a new home (moving in, Arranging the first ornaments, DIY, etc.).

View approved images (Trussle only)

Iconography

Within the product, icons are very functinal and help users understand the information they are viewing. For these purposes we use a simple line icon in Trussle Green with a line weight of 2px. Icons are linear and should have enough straight corners to reflect the logo marque.

At other times icons have a slightly different job. This is where illustration and iconography meet. We can use our illustration style to create icons in places that need to inspire, persuade or entertain users.

View our icons (Trussle only)
